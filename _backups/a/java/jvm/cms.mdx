CMS 收集器的全称是并发标记扫描（_Concurrent Mark Sweep_）收集器。

在 Java 9 中，它已经被标记为弃用状态了，详情见 [JEP 291](https://openjdk.org/jeps/291)。
也是在这个版本中，G1 成了默认收集器（见 [JEP 248](https://openjdk.org/jeps/248)）。

在 JDK 14 中，它被移除了。大致情况是，在 Java 9 之后，又引入了 ZGC 和 Shenandoah GC，
此后也没有人愿意维护 CMS 了，干脆删除之。详情见 [JEP 363](https://openjdk.org/jeps/363)。

总之，别在这上面浪费时间了。

如果你对其念念不忘的话，可以看看
[Getting Started with the G1 Garbage Collector - Oracle](https://www.oracle.com/webfolder/technetwork/tutorials/obe/java/G1GettingStarted/index.html)。
