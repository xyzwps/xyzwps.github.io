概要和详情

有时，查询概要和详情所付出的代价差别巨大。

在 UI 上，有时会有一个列表页面查询每项业务数据的概要，然后通过点击这些项再去查看每一项的详情数据。如果在设计 GraphQL schema 时，把详情相关字段挂到摘要 schema 中，就会出现管不住 API 使用者的情况，一不小心查出了当前页面的非必要数据，导致查询性能极差。
