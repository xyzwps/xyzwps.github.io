不同路径 II

题目63. 不同路径 II。

这一题和62. 不同路径相比，多了障碍物。而障碍物跟棋盘没关系，所以推不出所谓的数学通项公式。不过那题 DP 的思路可以直接搬过来，和不同路径相比，这题 dp[i][j]dp[i][j]dp[i][j] 多了一种可能： 如果 (i,j)(i,j)(i,j) 上放有障碍物，那么从 (i,j)(i,j)(i,j) 到达 (0,0)(0,0)(0,0) 处的路径数为 0。

这样，我们把不同路径那题的代码改改就可以继续用了。代码如下：

function uniquePathsWithObstacles(obstacleGrid: number[][]): number {
    const m = obstacleGrid.length;
    const n = obstacleGrid[0].length;
    const dp: number[][] = [];

    for (let i = 0; i < m; i++) {
        dp[i] = []
        for (let j = 0; j < n; j++) {
            if (i == 0 && j == 0) {
                dp[0][0] = 1 - obstacleGrid[0][0] // 处理终点
            } else if (obstacleGrid[i][j] == 1) {
                dp[i][j] = 0; // 有障碍物，无法到达终点
            } else if (i == 0) {
                dp[i][j] = dp[i][j - 1]; // 在边上
            } else if (j == 0) {
                dp[i][j] = dp[i - 1][j]; // 在边上
            } else {
                dp[i][j] = dp[i][j - 1] + dp[i - 1][j]
            }
        }
    }
    return dp[m - 1][n - 1]
};
