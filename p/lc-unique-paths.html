题目62. 不同路径。

DP 解法

设棋盘大小是 m×nm \times nm×n，其中 m>0,n>0m \gt 0, n \gt 0m>0,n>0。设 dp[i][j]dp[i][j]dp[i][j]（提前暴露） 表示从大小为 i×ji \times ji×j 的棋盘左上角走到右下角的走法。显然，本地的答案是 dp[m][n]dp[m][n]dp[m][n]。

下面我们考虑如何求任意一个 dp[i][j]dp[i][j]dp[i][j]，其中 m≥i>0,n>j>0m \ge i \gt 0, n \gt j \gt 0m≥i>0,n>j>0。 因为题目没说左上角和右下角的坐标是什么，这里为了方便，令左上角的坐标是 (m,n)(m,n)(m,n)，右下角的坐标是 (1,1)(1,1)(1,1)。 显然，从坐标 (i,j)(i,j)(i,j) 处走到 (1,1)(1,1)(1,1) 处的走法数量，就是 dp[i][j]dp[i][j]dp[i][j]。

首先，dp[1][j]=1,dp[i][1]=1dp[1][j] = 1, dp[i][1] = 1dp[1][j]=1,dp[i][1]=1，对所有的 m≥i>0,n>j>0m \ge i \gt 0, n \gt j \gt 0m≥i>0,n>j>0 成立， 因为在棋盘边缘没有办法转弯，所以只有一种走法。如果 i>1i \gt 1i>1 且 j>1j \gt 1j>1，那么从格子 (i,j)(i,j)(i,j) 处起步，有两种走法——横着走和竖着走，而且这两种走法走一步之后就到了 (i−1,j)(i-1,j)(i−1,j) 或者 (i,j−i)(i,j-i)(i,j−i)，这就是在说， dp[i][j]=dp[i−1][j]+dp[i][j−1]dp[i][j] = dp[i-1][j] + dp[i][j-1]dp[i][j]=dp[i−1][j]+dp[i][j−1]。

沿着这样的思路，就可以写出下面的代码（注意，分析中棋盘右下角的坐标是 (1,1)(1,1)(1,1)，但是代码里的二维数组坐标是从 右下角的坐标是 (0,0)(0,0)(0,0) 开始的）：

function uniquePaths(m: number, n: number): number {
    const dp: number[][] = [];
    for (let i = 0; i < m; i++) {
        dp[i] = []
        for (let j = 0; j < n; j++) {
            if (i == 0 && j == 0) {
                dp[0][0] = 1;
            } else if (i == 0 || j == 0) {
                dp[i][j] = 1;
            } else {
                dp[i][j] = dp[i][j - 1] + dp[i - 1][j]
            }
        }
    }
    return dp[m - 1][n - 1]
};

公式法

我们注意到，函数 uniquePaths 是一个关于 mmm 和 nnn 的纯函数，这意味着我们可以推出通项公式。 LeetCode 官方题解说得很好，我直接抄过来了。

从左上角到右下角的过程中，我们需要移动 m+n−2m+n−2m+n−2 次，其中有 m−1m−1m−1 次向下移动，n−1n−1n−1 次向右移动。因此路径的总数，就等于从 m+n−2m+n−2m+n−2 次移动中选择 m−1m−1m−1 次向下移动的方案数，即组合数：
